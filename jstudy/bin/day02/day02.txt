자바로 만든 프로그램을 실행하는 원리
	1. 자바언어 문법에 맞게 자연어로 프로그램을 작성(==> .java파일을 만든다.)
	
		작성한 프로그램은 저장한다
		규칙 
			1. 저장할 파일의 이름은 class의 이름과 반드시 동일.
			2. 확장자는 반드시 .java .
			
			 예 ] 
			 	public class Test{
			 	}
			 	==> Test.java 로 저장되어야 함
			 	
	2. 만들어진 소스파일은 컴파일을 해야한다.
		이렇게하면 자바가상머신이 읽을 수 있는 바이트코드가 만들어지고
		이 바이트코드를 기계어로 번역해서 컴퓨터가 실행.
		
		컴파일 방법
		
			javac 파일이름.java
			
			예 ]
				javac Test.java
				
			경우에따라 옵션이 붙기도한다.
		컴파일단계를 거치면 확장자가 .class인 파일이 생성
		==> Test.class
		
		참고 ]
			cmd창에서는 현재 폴더에 있는 도구만 사용가능.
			다른 폴더에 있는 도구를 사용하기 위해서 연결시켜줘야함.
			이 과정을 "path"에 등록한다. 라고 함.
			
	3. 생성된 .class 파일(바이트코드)은 배포됨.
		==>다른 사람에게 실행하도록 줄 수 있다.
		
		실행명령 ]
			java Test
			
			
	용어설명 ] 
		API 
		==> 운영체제와 프로그램 사이의 원활한 작동을 위해
			양쪽을 연결해주는 도구
			
		라이브러리
		==> API가 사용하는 도구에 필요한 정보를 기억한 프로그램의 일종
		
		우리가 JDK를 설치하면 자바로 만든 프로그램과 해당 운영체제 사이를 원활하게
		작동시키기 위한 API도 같이 설치됨.
		
		물론 이것에 포함하여 라이브러리도 같이 설치됨
		특정 명령을 수행하기 위해서는 어떤 절차를 거쳐야하는지 기록한 문서
		
	결론 ] 
		API내부에 라이브러리가 반드시 포함되어야함.

자바프로그램의 구조
	자바프로그램은 반드시 모든 내용이 하나의 class안에 포함되어야함
	
	하나의 파일내부에
	class Test01{
	}			
	class Test01_01{
	}
	class Test01_02{
	}
	
	1) class를 만드는 방법
	
		형식 ] 
			[ 접근지정자 ] [속성] class 클래스이름 {
				클래스내용...
			}
		예 ]
			public class Test{
			
			}
			
		식별자
			자바에서 프로그램을 만들다보면 개발자가 이름을 만들어야하는 경우가 발생.
			개발자가 이름을 붙이는 것을 "사용자 정읭어 || 식별자" 라고 한다.
			
			종류 ]
				클래스이름, 함수이름, 변수이름, ...
			
			규칙 ]
				1. 첫글자는 알파벳으로 시작.
					클래스의 경우는 소문자로 시작해도 되지만 
					암묵적으로 대문자로 만들어주는게 좋다.
					(첫글자가 숫자로 시작하면 절대안됨)
				2. 두번째 글자부터는 문자와 숫자 그리고 $,_ 를 혼용 가능
				3. 특수문자는 _,$ 만 사용가능 (첫 문자로도 사용가능)
				4. 식별자의 길이는 256글자 이내로.
				
				예 ] 
					1hong ---> x
					hong1 ---> o
			
			식별자 표기법
				1. 카멜표기법 : 식별자가 두 단어 이상으로 된경우 두번째 단어의 첫 문자는 대문자
				2. 스네이크표기법 : 두 단어 이상인 경우 각 단어사이에 _를 이용, 연결하는 방법
				
	2) 파일 이름을 지정하는 방법
		
		1. 하나의 파일에는 하나의 class가 작성되는 것을 원칙
			따라서 파일의 이름은 class의 이름과 동일하게 만들어야함.
		2. 어쩔 수 없이 하나의 파일에 두개 이상의 class가 만들어지게 되는 경우
			
			(1) class 중 public이 붙은 class를 파일이름으로 사용
				<== 하나의 파일에서 public이 붙어있는 class는 0 | 1개만 허락
			(2) public이 붙지 않은경우
				a. main 함수가 있는 class이름이로 파일이름을 사용.
				b. main 함수가 없거나 여러개인경우 아무 class나 파일이름으로 사용.
				
	3) class가 실행되기 위해서는 자바가상머신(JVM)에 의해서 실행되는 함수가 있어야함
		자바가상머신은 아무함수나 사용하지않고 
		오직
			"public static void main(String[] args){}"
		함수만 호출한다.
		
		만약 어떤 class를 제작하고, 실행되기 위해서는 반드시 public static void main(String[] args) 함수가 존재해야함.
		이처럼 어떤 프로그램을 시작하는 함수를 '진입점 함수'라고 부른다.
		
	4) class안에는 필요한 함수나 필요한 변수를 포함가능
	
	결론]
		public class Test{
			int no;
			String name;
			public double getMoney(){
			}
			public void saveMoney(int n){}
			
			public static void main(String[] args){
			
			}
			
		}
